single server

Git push to production in <5 seconds.

The easiest way to deploy multiple GitHub repos on a single server with Cloudflare, Tailscale, and Kamal. All you need is a single Linux server.

Get going

Every command runs on your server.

SSH to your Linux server:

ssh root@<server_ip>

Set up your server:

curl -fsSL https://singleserver.com/install.sh | sh

Add your apps:

singleserver add https://github.com/you/my-app-1
singleserver add https://github.com/you/my-app-2
singleserver add https://github.com/you/my-app-3

What happens next

Whenever you push to GitHub, your server fetches the commit, builds it locally, and uses Kamal to roll it out with zero downtime. Each app runs isolated in its own Docker container.

git push → GitHub sends a webhook to your server
build → Your server builds the image from that commit
live → Kamal rolls it out with zero downtime

How it works

Install once on your server.

The installer prepares the machine with Docker, Kamal, Tailscale, cloudflared, a local Docker registry, a deploy user, and the Single Server daemon. The daemon runs under systemd, listens locally, and keeps its state in /etc/singleserver.

Connect the outside world.

Cloudflare Tunnel exposes the small public surface your server needs: GitHub webhooks, setup pages, and app domains. Tailscale gives you private SSH access to the machine. GitHub sends signed push webhooks to your server, and the GitHub App gives Single Server short-lived tokens to fetch code and report commit status.

Add repos from the host.

singleserver add checks that the GitHub App can read the repo, infers the app name and default domain, writes the app to /etc/singleserver/apps.yml, and configures Cloudflare DNS and tunnel routes for the app.

Build and deploy locally.

On each push, Single Server verifies the webhook, checks the repo against apps.yml, fetches the exact pushed commit, and asks Kamal to build the Docker image on the same server. Images go through the local registry, so deploys avoid an external CI queue and registry round trip.

Swap traffic safely.

Kamal starts the replacement container, waits for the healthcheck, and only then moves traffic to the new version. App logs, deploy logs, domains, server-side env vars, persistent storage, SQLite-friendly backups, and diagnostics stay available from the same singleserver CLI.

What you get

Fast deploys

No Actions queue, no per-repo runner hop. Build on the server, swap containers, healthcheck, and set commit status.

App domains

Domains are inferred or added when you add apps. Cloudflare Tunnel hides your server IP, handles public TLS, and keeps CDN and proxy features on by default.

Tiny Go tool

No Node runtime, no database, no per-repo runner. A small Go binary manages deploys from your server.

Containers per app

Every project runs in its own Docker container, but the host still stays simple and inspectable.

Kamal without boilerplate

Single Server generates deploy config from conventions, while Kamal handles the container rollout.

Monitor from the host

Use Tailscale SSH, status, logs, and doctor commands from the same server where deploys run.
